VIEW — a saved query with a name
A VIEW stores a SELECT statement under a name. It behaves like a table but holds no data — it reruns the query every time you use it.
CREATE VIEW it_staff AS SELECT name, salary FROM employees WHERE dept = 'IT'; -- Use it exactly like a real table: SELECT * FROM it_staff; SELECT * FROM it_staff WHERE salary > 5500; -- Remove it (data stays safe): DROP VIEW it_staff;

When to use a VIEW:
Hide sensitive columns from certain users (e.g. salaries) Avoid repeating a long, complex JOIN every time Keep your SQL clean and reusable
